## 5. 本地启动步骤

### 5.1 启动 PostgreSQL

```bash
export PATH="/d/postsql/postgresql-18.4-1-windows-x64-binaries/pgsql/bin:$PATH"
pg_ctl -D /d/postsql/postgresql-18.4-1-windows-x64-binaries/pgsql/data -l /d/postsql/pg.log start
```

### 5.2 初始化数据库（首次）

```bash
export PATH="/d/postsql/postgresql-18.4-1-windows-x64-binaries/pgsql/bin:$PATH"
initdb -U postgres --encoding=UTF8 --locale=C
pg_ctl -D /d/postsql/postgresql-18.4-1-windows-x64-binaries/pgsql/data start
psql -U postgres -c "CREATE USER znl WITH PASSWORD 'znl123';"
psql -U postgres -c "CREATE DATABASE znl_db OWNER znl;"
```

### 5.3 启动后端

```bash
cd backend
conda run -n ai_dev uvicorn app.main:app --host 0.0.0.0 --port 8000
```

首次启动会自动建表，之后运行 seed 创建管理员账号：

```bash
conda run -n ai_dev python -m app.seed
```

### 5.4 启动前端

```bash
cd frontend
npm install
npx vite --host 0.0.0.0 --port 3000
```

### 5.5 访问

- 前端页面：http://localhost:3000
- API 文档：http://localhost:8000/docs
- 默认账号：`admin` / `admin123`

## 6. 常见问题

### 端口被占用

```bash
netstat -ano | grep LISTENING
taskkill -f -pid <PID>
```

### 数据库连不上

检查 PostgreSQL 是否启动：

```bash
export PATH="/d/postsql/postgresql-18.4-1-windows-x64-binaries/pgsql/bin:$PATH"
pg_isready
```

### pip 安装报代理错误

Python 检测到系统代理 `127.0.0.1:10808`，如果该代理未运行则 pip 无法联网。
解决方案：使用 conda 环境 `ai_dev` 而非直接 pip install。
