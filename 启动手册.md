# 智农粮系统启动手册

本文记录本机项目、数据库、后端、前端的路径和启动命令。

## 1. 项目路径

项目根目录：

```text
D:\Desktop\my
```

后端目录：

```text
D:\Desktop\my\backend
```

前端目录：

```text
D:\Desktop\my\frontend
```

## 2. 数据库路径和命令

PostgreSQL 根目录：

```text
D:\postsql
```

PostgreSQL 程序目录：

```text
D:\postsql\postgresql-18.4-1-windows-x64-binaries\pgsql\bin
```

数据库数据目录：

```text
D:\postsql\data
```

日志文件：

```text
D:\postsql\logfile
```

启动数据库：

```powershell
D:\postsql\postgresql-18.4-1-windows-x64-binaries\pgsql\bin\pg_ctl.exe start -D D:\postsql\data -l D:\postsql\logfile
```

如果提示端口 5432 已经被占用，先执行下面的“测试数据库连接”。只要能返回 `1`，说明数据库已经在运行，不需要重复启动。

查看数据库状态：

```powershell
D:\postsql\postgresql-18.4-1-windows-x64-binaries\pgsql\bin\pg_ctl.exe status -D D:\postsql\data
```

停止数据库：

```powershell
D:\postsql\postgresql-18.4-1-windows-x64-binaries\pgsql\bin\pg_ctl.exe stop -D D:\postsql\data
```

测试数据库连接：

```powershell
D:\postsql\postgresql-18.4-1-windows-x64-binaries\pgsql\bin\psql.exe -h 127.0.0.1 -p 5432 -U znl -d znl_db -c "select 1;"
```

数据库连接配置在：

```text
D:\Desktop\my\backend\.env
```

当前配置：

```text
DB_HOST=localhost
DB_PORT=5432
DB_USER=znl
DB_PASSWORD=znl123
DB_NAME=znl_db
```

数据库当前只监听本机地址，账号连接使用 `scram-sha-256` 密码认证。这个设置适合自用：本机前后端可以访问，局域网其他电脑不能直接连数据库。

## 3. 后端环境和命令

后端虚拟环境：

```text
D:\Desktop\my\backend\.venv
```

后端 Python：

```text
D:\Desktop\my\backend\.venv\Scripts\python.exe
```

后端只允许本机前端来源访问 API，默认允许：

```text
http://127.0.0.1:5173
http://localhost:5173
```

对应配置在：

```text
D:\Desktop\my\backend\app\core\config.py
```

第一次配置依赖，或依赖变更后执行：

```powershell
cd D:\Desktop\my\backend
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

启动后端：

```powershell
cd D:\Desktop\my\backend
.\.venv\Scripts\python.exe -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8001
```

后端接口文档：

```text
http://127.0.0.1:8001/docs
```

如果 8001 端口被占用，先关闭旧的后端窗口，或换端口启动。换端口后，也要同步修改 `D:\Desktop\my\frontend\vite.config.ts` 里的代理地址。

```powershell
cd D:\Desktop\my\backend
.\.venv\Scripts\python.exe -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8002
```

## 4. 前端环境和命令

前端目录：

```text
D:\Desktop\my\frontend
```

前端依赖目录：

```text
D:\Desktop\my\frontend\node_modules
```

启动前端：

```powershell
cd D:\Desktop\my\frontend
npm.cmd run dev -- --host 127.0.0.1
```

前端访问地址：

```text
http://127.0.0.1:5173/
```

如果 5173 端口被占用，可以换端口：

```powershell
cd D:\Desktop\my\frontend
npm.cmd run dev -- --host 127.0.0.1 --port 5174
```

## 5. 每次正常启动顺序

建议开三个 PowerShell 窗口。

第一个窗口：启动数据库。

```powershell
D:\postsql\postgresql-18.4-1-windows-x64-binaries\pgsql\bin\pg_ctl.exe start -D D:\postsql\data -l D:\postsql\logfile
```

第二个窗口：启动后端。

```powershell
cd D:\Desktop\my\backend
.\.venv\Scripts\python.exe -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8001
```

第三个窗口：启动前端。

```powershell
cd D:\Desktop\my\frontend
npm.cmd run dev -- --host 127.0.0.1
```

最后打开：

```text
http://127.0.0.1:5173/
```

## 6. 数据安全和备份

PostgreSQL 默认开启断电恢复所需的 WAL 机制，当前配置中的 `fsync`、`synchronous_commit`、`full_page_writes` 都保持默认安全值。正常情况下，突然断电后再次启动数据库，PostgreSQL 会自动用 WAL 恢复到一致状态。

但断电保护不等于永远不会丢数据。硬盘损坏、误删、数据库文件损坏、病毒或人为操作都可能导致数据丢失，所以必须做备份。

备份目录：

```text
D:\postsql\backup
```

手动备份数据库：

```powershell
cd D:\Desktop\my\backend
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\backup.ps1
```

备份文件格式：

```text
D:\postsql\backup\znl_年月日_时分秒.sql
```

建议每天收工后执行一次备份。如果当天录入了很多数据，中途也可以手动备份一次。

设置每天自动备份。下面命令会创建一个 Windows 计划任务，每天 18:30 自动备份一次：

```powershell
schtasks /Create /TN "ZhiNongLiangDailyBackup" /SC DAILY /ST 18:30 /TR "powershell.exe -NoProfile -ExecutionPolicy Bypass -File D:\Desktop\my\backend\scripts\backup.ps1"
```

查看自动备份任务：

```powershell
schtasks /Query /TN "ZhiNongLiangDailyBackup"
```

手动运行一次自动备份任务：

```powershell
schtasks /Run /TN "ZhiNongLiangDailyBackup"
```

删除自动备份任务：

```powershell
schtasks /Delete /TN "ZhiNongLiangDailyBackup"
```

恢复数据库前要非常谨慎：恢复会把备份里的数据写回数据库。建议先复制一份最新备份文件，再操作。

恢复命令示例：

```powershell
D:\postsql\postgresql-18.4-1-windows-x64-binaries\pgsql\bin\psql.exe -h 127.0.0.1 -p 5432 -U znl -d znl_db -f D:\postsql\backup\你的备份文件.sql
```

## 7. 自用安全注意事项

平时启动后端时保持这个地址：

```powershell
--host 127.0.0.1
```

不要改成：

```powershell
--host 0.0.0.0
```

`127.0.0.1` 只允许本机访问，别人不能从局域网直接访问你的后端。`0.0.0.0` 会把服务暴露给同一网络里的其他设备，风险更高。

前端也建议使用：

```powershell
npm.cmd run dev -- --host 127.0.0.1
```

不要把 `.env`、数据库备份、登录测试文件发给别人，也不要上传到公开仓库。
